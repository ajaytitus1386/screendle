  <name>Task 1: Define CRT palette and dark theme in app.css</name>
  <files>src/app.css</files>
  <action>
Modify src/app.css to add the CRT color palette and warm dark theme:

1. **Add @font-face declaration** before the @theme block:
```css
@font-face {
  font-family: 'Audiowide';
  src: url('/fonts/audiowide/audiowide-regular.woff2') format('woff2');
  font-weight: 400;
  font-style: normal;
  font-display: swap;
}
```

2. **Add CRT palette colors to a NEW @theme block** (separate from the existing `@theme inline` block). This new block defines CRT-specific tokens. Place it AFTER the `@custom-variant dark` line and BEFORE the `:root` block:
```css
@theme {
  /* CRT phosphor colors */
  --color-crt-amber: oklch(0.69 0.14 56);     /* #faa622 */
  --color-crt-yellow: oklch(0.93 0.15 97);    /* #ffe52c */
  --color-crt-cyan: oklch(0.87 0.09 211);     /* #7fe6ef */
  --color-crt-lime: oklch(0.81 0.14 108);     /* #c4d70c */
  --color-crt-red: oklch(0.45 0.19 25);       /* #c22303 */

  /* Dark theme foundation */
  --color-dark-warm: oklch(0.15 0.01 56);     /* Warm dark ~#1c1a18 */
  --color-dark-surface: oklch(0.18 0.01 56);  /* Slightly lighter surface for cards */
  --color-text-cream: oklch(0.95 0.01 56);    /* Warm cream ~#f5f0e8 */

  /* Headline font */
  --font-headline: 'Audiowide', 'Impact', 'Arial Black', sans-serif;
}
```

3. **Update the `.dark` block** to use warm CRT-themed colors:
- `--background`: change to `oklch(0.15 0.01 56)` (warm dark, not cold blue)
- `--foreground`: change to `oklch(0.95 0.01 56)` (warm cream text)
- `--card`: change to `oklch(0.18 0.01 56)` (slightly lighter warm surface)
- `--card-foreground`: change to `oklch(0.95 0.01 56)` (warm cream)
- `--popover`: same as card
- `--popover-foreground`: same as card-foreground
- `--muted-foreground`: change to `oklch(0.65 0.02 56)` (warm muted, not cold blue)
- `--destructive`: change to `oklch(0.45 0.19 25)` (match CRT red)
- `--border`: change to `oklch(0.69 0.14 56 / 15%)` (amber-tinted borders)
- `--input`: change to `oklch(0.69 0.14 56 / 20%)` (amber-tinted inputs)
- `--ring`: change to `oklch(0.87 0.09 211)` (cyan ring for focus, per user decision: cyan for interactive)

Keep remaining .dark variables as reasonable warm variants. The key principle: shift from cold blue-tinted oklch hues (264-265 degree range) to warm amber hues (~56 degrees) at similar lightness levels.

4. **Update scrollbar colors** in @layer base to use warm tones:
- thumb: `oklch(0.69 0.14 56 / 20%)` (amber tint)
- thumb:hover: `oklch(0.69 0.14 56 / 35%)`
- scrollbar-color: `oklch(0.69 0.14 56 / 20%) transparent`

IMPORTANT: Do NOT delete or restructure the existing `@theme inline` block (lines 76-112). That block maps shadcn CSS variables to Tailwind tokens and must remain. The new `@theme` block is separate and additive.

1. **Download Audiowide WOFF2 font file**:
```bash
mkdir -p static/fonts/audiowide
# Download from Google Fonts Helper or Fontsource
curl -L "https://fonts.gstatic.com/s/audiowide/v20/l7gdbjpo0cum0ckerWCdlg_O.woff2" -o static/fonts/audiowide/audiowide-regular.woff2
```
Verify the file is a valid WOFF2 (should be ~15-25KB). If the Google Fonts URL doesn't work, try fontsource: `npm install @fontsource/audiowide` then copy the woff2 file from `node_modules/@fontsource/audiowide/files/` to `static/fonts/audiowide/`.

2. **Update src/app.html** — add font preload in head before %sveltekit.head%:
```html
<head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />

    <!-- Preload critical Audiowide headline font -->
    <link rel="preload"
          href="/fonts/audiowide/audiowide-regular.woff2"
          as="font"
          type="font/woff2"
          crossorigin="anonymous" />

    %sveltekit.head%
</head>
```
CRITICAL: Include `crossorigin="anonymous"` — without it the font gets downloaded twice (pitfall from research).
