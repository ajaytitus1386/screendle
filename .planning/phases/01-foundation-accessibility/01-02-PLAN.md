**In src/app.css**, add the following CSS AFTER the @layer base block. Use a clear comment header to mark scanline code as isolated and removable:

```css
/* ========================================
   CRT SCANLINE OVERLAY
   Isolated layer — remove this entire block to disable scanlines.
   See: FOUN-03, FOUN-05
   ======================================== */

.crt-scanlines {
  position: relative;
}

.crt-scanlines::before {
  content: '';
  position: fixed;
  inset: 0;
  pointer-events: none;
  z-index: 9999;

  /* Amber-tinted horizontal scanlines
     - 4px spacing: visible as scanlines without moire
     - 6% opacity: noticeable subconsciously, not distracting
     - Warm amber tint matches CRT phosphor aesthetic */
  background: repeating-linear-gradient(
    0deg,
    oklch(0.69 0.14 56 / 6%) 0px,
    oklch(0.69 0.14 56 / 6%) 1px,
    transparent 1px,
    transparent 4px
  );
}

/* ========================================
   ACCESSIBILITY: REDUCED MOTION
   Comprehensive support for prefers-reduced-motion.
   See: FOUN-04
   ======================================== */

@media (prefers-reduced-motion: reduce) {
  /* Remove scanline overlay */
  .crt-scanlines::before {
    display: none;
  }

  /* Disable all animations and transitions globally */
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

Key decisions per user constraints:
- 6% opacity (within 5-8% range, Claude's discretion)
- oklch(0.69 0.14 56) = CRT amber color, provides warm tint not pure black
- 4px spacing for visible scanline texture without moire
- Static only — no animation or drift
- `position: fixed` with `inset: 0` covers viewport
- `pointer-events: none` ensures click-through
- `z-index: 9999` ensures overlay is above content

**In src/routes/+layout.svelte**, restructure the layout so scanlines appear on content area but NOT on the dark navbar:

Change the template structure from:
```svelte
<div class="dark min-h-screen bg-background text-foreground overflow-x-hidden">
  <!-- Navbar -->
  <nav ...>...</nav>
  {@render children()}
  <HowToPlay ... />
</div>
```

To:
```svelte
<div class="dark min-h-screen bg-background text-foreground overflow-x-hidden">
  <!-- Navbar - outside scanline container, no scanlines here -->
  <nav ...>...</nav>

  <!-- Content area with CRT scanline overlay -->
  <div class="crt-scanlines">
    {@render children()}
  </div>

  <HowToPlay ... />
</div>
```

The scanlines wrap only the content area, not the navbar. The HowToPlay modal is outside scanlines since it has its own overlay/backdrop. The `position: fixed` on the pseudo-element means the scanlines visually cover the full viewport when scrolling content, but the container element being after the nav means the stacking context is correct.

IMPORTANT: Do NOT move the nav inside `.crt-scanlines`. The user explicitly said "main content area only — not on the dark header/navbar."

NOTE: Since `position: fixed` makes the scanlines cover the full viewport regardless of container placement, and the user wants no scanlines on the navbar, we need to clip the scanlines. Add a `clip-path` or use `position: absolute` instead of `fixed` with the container having `min-height` to cover content. Actually, the simplest approach: use `position: fixed` for full coverage BUT add a negative top offset or clip for the navbar height. Alternatively, since the navbar is dark and scanlines are amber-tinted at 6% opacity, they'll be nearly invisible on the dark navbar anyway. However, to strictly follow the user's instruction, set the pseudo-element to `position: absolute` instead of `fixed`, and make the `.crt-scanlines` container `min-h-[calc(100vh-3rem)]` (3rem = h-12 navbar):

```css
.crt-scanlines {
  position: relative;
  min-height: calc(100vh - 3rem); /* Fill below navbar */
}

.crt-scanlines::before {
  content: '';
  position: absolute;
  inset: 0;
  pointer-events: none;
  z-index: 50; /* Above content, below modals */
  /* ... gradient stays the same ... */
}
```

Use `z-index: 50` (not 9999) so the overlay sits above page content but below modal dialogs/popovers which typically use z-index 50+.
